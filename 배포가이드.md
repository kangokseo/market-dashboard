# 시황 대시보드 GitHub Pages 배포 가이드

배포 대상 파일: `deploy/index.html` (자립형 단일 파일, 외부 의존성 없음)
배포 후 누구나 URL 링크만으로 접속 가능.

---

## 방법 A — 웹 브라우저만으로 (CLI 불필요, 가장 쉬움)

1. https://github.com 로그인 → 우측 상단 `+` → **New repository**
2. Repository name: 예) `market-dashboard` → **Public** 선택 → **Create repository**
3. 생성된 저장소 화면에서 **uploading an existing file** 클릭
4. `deploy/index.html` 파일을 드래그&드롭 → **Commit changes**
5. 저장소 상단 **Settings** → 좌측 **Pages**
6. *Branch* 를 `main` / `/ (root)` 으로 지정 → **Save**
7. 1~2분 후 같은 Pages 화면 상단에 공개 URL 표시:
   `https://<사용자명>.github.io/market-dashboard/`

이 URL을 공유하면 누구나 대시보드를 볼 수 있음.

---

## 방법 B — git CLI (반복 업데이트에 편리)

```bash
cd deploy
git init
git add index.html
git commit -m "deploy market dashboard"
git branch -M main
git remote add origin https://github.com/<사용자명>/market-dashboard.git
git push -u origin main
```

이후 GitHub 저장소 **Settings → Pages** 에서 Branch `main` / root 지정.

---

## 데이터 갱신 시

`agent_update.py` 로 데이터를 갱신한 뒤에는 임베드 값이 옛 데이터로 남으므로,
대시보드를 다시 자립형으로 빌드하고 `index.html` 을 교체 후 재업로드(또는 git push)해야 함.

## 참고
- 저장소를 **Private** 으로 두면 Pages 공개 URL이 동작하지 않을 수 있음(무료 플랜). 공유 목적이면 **Public** 권장.
- 데이터 노출이 부담되면 GitHub Pages 대신 비밀번호 보호가 가능한 Netlify/Cloudflare Pages 사용 고려.
